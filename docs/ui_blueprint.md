# 営業アプリ UI 設計書

## プロダクト原則

- アプリの中心はカレンダーそのものではなく、営業担当者の次のアクションに置く。
- 主要フローは必ず `予定 -> 顧客 -> 通話/面談 -> 記録 -> 同期` につながるようにする。
- 未記録の面談、同期失敗、フォロー漏れをすぐ見つけられる状態にする。

## 主要ナビゲーション

### 1. ホーム

目的:
- 今日対応すべきことを一目で把握できるようにする。
- 面談前後の操作回数をできるだけ減らす。

主なセクション:
- 今日のサマリー
- 直近の面談
- 記録待ちキュー
- 最近の通話と AI 要約
- 同期状況
- クイックアクション

ワイヤーフレーム:

```text
+--------------------------------------------------+
| おはようございます、Alex                         |
| 今日の面談 4件 | 記録待ち 2件 | アラート 1件    |
+--------------------------------------------------+
| [次の面談カード]                                 |
| 10:30 - 11:30  ACME / 東京本社                   |
| 「四半期更新ミーティング」                       |
| [経路] [電話] [記録を開く]                       |
+--------------------------------------------------+
| 記録待ち                                         |
| - 11:30 ACME / ジオフェンス離脱 / 未記録         |
| - 15:00 Beta / 終了5分後 / 下書きなし            |
+--------------------------------------------------+
| 最近のアクティビティ                             |
| - 通話要約: 田中 / 来週火曜にフォロー            |
| - 面談記録を Sheets に同期済み                   |
+--------------------------------------------------+
| クイックアクション                               |
| [名刺スキャン] [面談追加] [電話] [顧客追加]      |
+--------------------------------------------------+
```

### 2. カレンダー

目的:
- Google Calendar の CRUD を管理する。
- 予定を単なる日付データではなく、営業アクションの起点として扱う。

主なセクション:
- 月表示
- 日表示
- 予定詳細
- 予定作成/編集

ワイヤーフレーム:

```text
+--------------------------------------------------+
| 2026年3月                           [+ 予定追加] |
+--------------------------------------------------+
| カレンダーグリッド / タイムライン               |
+--------------------------------------------------+
| 選択日の予定一覧                                 |
| 09:00 社内定例                                   |
| 10:30 ACME / 東京本社 / 参加者3名                |
| 15:00 Beta / 品川 / フォローアップ               |
+--------------------------------------------------+
```

### 3. 面談詳細

目的:
- カレンダー予定を営業アクションのハブに変える。

セクション:
- 面談情報
- 顧客情報
- 場所と到着状況
- 録音とメモ操作
- 面談後リマインド設定

ワイヤーフレーム:

```text
+--------------------------------------------------+
| 四半期更新ミーティング                           |
| 10:30 - 11:30  東京本社                          |
| ステータス: 進行中                               |
+--------------------------------------------------+
| 顧客                                              |
| ACME Corp / 金 田中 / 戦略アカウント             |
| 最終接点: 5日前                                  |
+--------------------------------------------------+
| アクション                                        |
| [経路案内] [電話] [録音開始] [メモ作成]          |
+--------------------------------------------------+
| リマインドルール                                  |
| 終了後 +5分                                       |
| 場所から 300m 離脱                                |
+--------------------------------------------------+
| タイムライン                                      |
| チェックイン / 音声メモ / 下書き / 同期ログ       |
+--------------------------------------------------+
```

### 4. 面談記録フォーム

目的:
- 1分以内で意味のある営業記録を残せるようにする。

セクション:
- 要約
- 決裁者
- 顧客ニーズ
- リスク
- 次のアクション
- 同期状態

ワイヤーフレーム:

```text
+--------------------------------------------------+
| 面談記録                                         |
+--------------------------------------------------+
| 要約                                              |
| [ 複数行入力 ]                                    |
| 要点                                              |
| [ チップ / 箇条書き入力 ]                         |
| 次のアクション                                    |
| [ テキスト ] [ 期限 ] [ 担当者 ]                 |
| フォローアップ                                    |
| [ 次回予定を作成 ]                                |
+--------------------------------------------------+
| [下書き保存]                     [保存して同期]   |
+--------------------------------------------------+
```

### 5. 顧客

目的:
- 顧客企業、担当者、名刺取り込みを一元管理する。

セクション:
- 検索とフィルタ
- 顧客一覧
- 顧客詳細
- 連絡先一覧
- 名刺スキャン確認

ワイヤーフレーム:

```text
+--------------------------------------------------+
| 顧客                           [名刺スキャン] [追加]|
+--------------------------------------------------+
| 検索: [ ACME                         ]            |
| フィルタ: 稼働中 / 案件中 / 最近接点あり          |
+--------------------------------------------------+
| ACME Corp                                         |
| 連絡先3件 | 最終面談: 昨日                        |
| Beta Inc                                          |
| 連絡先1件 | フォロー期限超過                      |
+--------------------------------------------------+
```

### 6. アクティビティ

目的:
- 面談記録、通話履歴、録音、要約、通知履歴を時系列で集約する。

セクション:
- 統合タイムライン
- 種別フィルタ
- 失敗項目

ワイヤーフレーム:

```text
+--------------------------------------------------+
| アクティビティ                                    |
| フィルタ: すべて / 面談 / 通話 / 同期 / 通知      |
+--------------------------------------------------+
| 16:45  面談記録を Sheets に同期                   |
| 15:38  通話要約を作成                             |
| 11:35  面談記録リマインド送信                     |
| 11:32  面談場所のジオフェンス離脱                 |
+--------------------------------------------------+
```

### 7. その他

目的:
- 利用頻度は低いが重要な設定をまとめる。

セクション:
- Okta SSO アカウント
- Google Calendar 連携
- Google Sheets 連携
- 通知権限
- 録音ポリシーと同意
- 管理者向け機能

## 共通 UX ルール

- 各面談カードでは強い CTA を 1 つだけ見せる。
- 面談後の既定 CTA は `記録を書く` にする。
- 同期状態は入力した画面上で確認できるようにする。
- 位置トリガー通知は、なぜ発火したかを明示する。
- OCR や AI 抽出に不確実性がある場合は、保存前に確認シートを出す。
